# Configuration Reference

This page documents all configuration files used by pyArgos.

---

## Datasources Configuration

**Location:** `<experiment>/runtimeExperimentData/Datasources_Configurations.json`

The main configuration file for an experiment.

```json
{
    "experimentName": "MyExperiment",
    "kafka": {
        "bootstrap_servers": ["127.0.0.1:9092"]
    },
    "Thingsboard": {
        "restURL": "http://localhost:8080",
        "username": "tenant@thingsboard.org",
        "password": "tenant"
    }
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `experimentName` | Yes | Name of the experiment |
| `kafka.bootstrap_servers` | For Kafka features | List of Kafka broker addresses |
| `Thingsboard.restURL` | For TB features | ThingsBoard REST API URL |
| `Thingsboard.username` | For TB features | ThingsBoard login username |
| `Thingsboard.password` | For TB features | ThingsBoard login password |

## Experiment Data

**Location:** `<experiment>/experimentData/ExperimentData.json`

Defines the entities in the experiment.

```json
{
    "properties": {
        "calculationWindows": [60, 180]
    },
    "Entities": [
        {
            "entityType": "DEVICE",
            "Number": 3,
            "Type": "Sensor",
            "nameFormat": "Sensor_{id:02d}"
        }
    ]
}
```

| Field | Description |
|-------|-------------|
| `properties.calculationWindows` | Time windows (seconds) for shadow calculation devices |
| `Entities[].entityType` | `"DEVICE"` or `"ASSET"` |
| `Entities[].Number` | Number of entities to create |
| `Entities[].Type` | Entity type name |
| `Entities[].nameFormat` | Name template with `{id}` placeholder |

## Consumer Configuration

**Location:** `consumersConf.json`

Configures Kafka consumer behavior per topic.

```json
{
    "Sensor": {
        "slideWindow": "60",
        "processesConfig": {
            "None": {
                "argos.kafka.processes.to_parquet_CampbellBinary": {}
            },
            "180": {
                "argos.kafka.processes.calc_wind": {}
            }
        }
    }
}
```

| Field | Description |
|-------|-------------|
| `slideWindow` | Window size in seconds for sliding window processing |
| `processesConfig` | Map of window sizes to processing functions |

## Trial Design Files

**Location:** `<experiment>/experimentData/trials/design/<trialName>.json`

Per-entity trial configuration.

```json
{
    "Sensor_01": {
        "Name": "Sensor_01",
        "Type": "Sensor",
        "entityType": "DEVICE",
        "attributes": {
            "location": {"lat": 32.0, "lon": 34.8},
            "threshold": 25.0
        },
        "contains": [
            ["DEVICE", "SubSensor_01"]
        ]
    }
}
```

## Logging Configuration

**Location:** `~/.pyargos/log/argosLogging.config`

Standard Python logging configuration file. pyArgos defines a custom `EXECUTION` log level (15) between DEBUG (10) and INFO (20).

If this file doesn't exist, pyArgos uses a default configuration that logs to the console.

---

## Device Map

**Location:** `<experiment>/runtimeExperimentData/deviceMap.json`

Auto-generated by the Node-RED device map command.

```json
{
    "Sensor 1": {
        "entityType": "DEVICE",
        "entityName": "Sensor_0001"
    }
}
```

Maps simplified device identifiers to their full entity information for Node-RED routing.
